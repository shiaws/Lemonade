# Contributing to Lemonade 🍋

First off — thank you. Lemonade exists because people like you believe hardware shouldn't die on a corporate schedule.

---

## Where We Need Help

We are in early development and actively recruiting contributors. Here's where you can make the biggest impact:

### 🔧 Systems / Low Level (C/C++)
- ARM to x86 system call translation
- Kernel level compatibility work
- macOS internals and syscall mapping
- Performance and stability work

### 🍎 macOS / Swift
- Installer and setup UI
- macOS-facing application layer
- User experience and onboarding

### 🧪 Testing
- Intel Mac compatibility testing
- Hackintosh environment testing
- Bug reporting and reproduction

### 📝 Documentation
- Technical writing
- Architecture documentation
- Guides for end users and contributors

---

## Getting Started

1. **Fork** the repository
2. **Clone** your fork locally
3. Check the **Issues** tab for tasks tagged `good first issue`
4. Open a **Pull Request** with a clear description of what you changed and why

---

## Guidelines

- Be respectful — this is a community project
- Keep pull requests focused — one thing at a time
- Write clear commit messages
- If you're tackling something big, open an Issue first to discuss the approach before writing code
- All contributions must be compatible with **AGPL-3.0**

---

## Reporting Bugs

Open an Issue and include:
- Your macOS version
- Your hardware (Intel Mac model or Hackintosh specs)
- Steps to reproduce
- What you expected vs what happened

---

## Questions?

Open an Issue tagged `question` — no question is too small.
